  + GET получает данные GET/api/users ; безопасный, идемпатентен
идемпатентность - это когда можно повторять несколько раз без вреда(не меняет данные)

  -  POST создает новый объект (меняет состояние) POST/api/user меняет данные не безопасны и не идемпотантен
(если отправить два раза то создаст два пользователя два раза)

  + PUT обновление объекта (замена) PUT/api/user меняет данные , не безопасен , идемпатентен

  +- PATCH позволяют частично обновить объект,
не нужно отправлять весть объект(не все api поддерживают это), идемпатентный и да и нет
(так как данные поменяются только при первом запросе, при последующих уже нет)

  + DELETE удаление объкта(по идентификатору) идемпотентен действие необратимое

200 ok все успешно
201 Created объект успешно создан POST
400 Bad Request ошибка в данных
404 Not found объект не найден
500 Internal Server Error ошибка сервера
